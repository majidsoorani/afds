# Contributing to AFDS

## Getting Started

- Fork the repository.
- Create a feature branch from `main`.
- Make small, focused changes.
- Add or update tests when behavior changes.
- Open a pull request with a clear description.

## Pull Request Checklist

- Code builds and tests pass locally.
- API or behavior changes are documented.
- No secrets or private data are committed.
- New dependencies are justified.

## Data and Privacy Requirements

- Do not commit customer or production-derived data.
- Use synthetic/anonymized fixtures only.
- Respect third-party data provider terms.

## Review Expectations

Maintainers prioritize correctness, safety, and operational impact.
Large changes may require design notes before merge.
