# Security Policy

## Reporting a Vulnerability

Please do not open public issues for sensitive security vulnerabilities.

Report privately to the maintainers with:

- Description of the issue
- Steps to reproduce
- Impact assessment
- Suggested mitigation (if known)

## Response Targets

- Initial acknowledgment: within 3 business days
- Triage decision: within 7 business days
- Patch target for high severity: as soon as practical

## Scope

This project handles financial risk workflows. Treat potential leaks of secrets,
PII, auth bypass, and remote execution as high severity.
